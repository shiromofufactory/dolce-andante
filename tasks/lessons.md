# 教訓・修正パターンの記録

## L-001｜キャラ間の上下関係（学年・敬語）を意識する

**修正例**：まーちゃん（2年）→ふれんち（3年）の呼称・口調が「ふれんち君／タメ口」になっていた。

**正しい扱い**：
- まーちゃんは後輩なので、本来は **「ふれんちさん」＋丁寧語（〜ですか／〜ます）** が自然
- ほんわか系の性格でも、敬語は崩さないのが普通

**チェックリスト（脚本・追加台詞時）**：
- [ ] 話者と相手の学年差を確認したか？
- [ ] 後輩→先輩は「さん」付け＋丁寧語
- [ ] 同期同士はあだ名＋タメ口で OK
- [ ] 先輩→後輩は呼び捨て or 「ちゃん／君」＋タメ口
- [ ] 既に同期で親しい間柄が確立してる例外（例：ミオ↔ふれんち、薫↔ふれんち）はそのまま

**学年マップ（参考）**：
- 4年：ミヤコさん、加藤さん（25歳・社会人）
- 3年：ふれんち、ミオ、薫、ユタカ、平野、姫、シェリー、長友、東山、たえやん、愛ちゃん、中野さん、ヨコノ
- 2年：渡部、まーちゃん
- 1年：ヒマリちゃん

## L-002｜キャラの外見・髪型・服装の変化を描写するときは、立ち絵と矛盾させない

**修正例**：3-01 で「ヒマリちゃんは何故かツインテールをやめてポニーテールになっていた」と描写していたが、実際の立ち絵はずっとポニーテール。プレイヤーが「あれ、ツインテールだったっけ？」と混乱する。

**正しい扱い**：
- 立ち絵に固定された外見の特徴（髪型・服装の決定的な要素）は変化させない
- 季節感・時間経過は **「雰囲気がほんの少しだけ大人っぽく」「日焼けしている」「ちょっと痩せた」** など、立ち絵で表現されない／表現されても誤差レベルの曖昧な描写にする
- 服装変更を匂わせるなら、ステージ衣装・特別衣装は CG 化前提

**チェックリスト（脚本・追加台詞時）**：
- [ ] そのキャラの立ち絵の特徴（髪型、服装、メガネの有無、肌の色）と矛盾していないか？
- [ ] 「変化した」と書く場合、立ち絵で表現できない種類の変化か？
- [ ] 確実な変化として描写するなら、CG として用意できるか？

## L-003｜キャラ設定（性別・口調・性格）と発言・描写の矛盾に注意

**修正例**：「店長は無口なおじいちゃん」と設定しているのに、別シーンで「店長が『うちの店、今日は貸し切り状態ね』と笑ってくれた」という女性的口調＋饒舌な描写が入っていた。

**正しい扱い**：
- 一度確立したキャラ設定（無口／寡黙／頑固など）を崩す描写は入れない
- 端役でも発言口調（女性的な終助詞「ね」「わ」など）が性別と矛盾しないか確認
- 無口系キャラは「黙って頷く」「奥に引っ込む」など行動で示す

## L-004｜描写と実際の登場人物が一致しているか確認

**修正例**：「見慣れない組み合わせが通りかかった」と書いたが、実際に登場するのは１人だけ。元案（複数人）から変更された際に文言が残ってしまっていた。

**チェックリスト**：
- [ ] 「組み合わせ」「複数」「二人」等の数を示す語と、実際に表示されるキャラ数が一致しているか？
- [ ] 「珍しい」「見慣れない」等の形容を使うとき、その理由が文脈から読み取れるか？

## L-005｜本番／打ち上げ等、中野さんのような「選択的参加」キャラの配置

**修正例**：「中野さんは、当然、俺たちの隣にはいなかった」と書いた直後に「彼女は、本番だけは、絶対に来る」と続き、文章として破綻していた。

**正しい扱い**：
- 「本番には来るが打ち上げには来ない」というキャラ設定があるなら、本番シーンでは「ちゃんと弾いていた」等、打ち上げシーンでは「いない」と、シーンごとに正しく描写する

## L-006｜分岐フラグ／選択肢によって成立する文言は、必ず分岐側で出し分ける

**修正例**：ミオからの「さっき、言いそびれた」「誘ったら来てくれる？」というメール文面は、飲み会でミオの隣に行って特定の選択肢を選んだ場合にのみ成立する前振り。しかし、この選択をしていないルートでも同じメールが届く仕様になっていた。

**正しい扱い**：
- 前振り選択肢を通ったときに専用フラグ（例：`mio_invite_hinted`）を立てる
- 後続シーンで `condJump` により分岐し、前振りを踏まえた文面と、踏まえない文面を用意する

**チェックリスト**：
- [ ] メールや回想で「さっき」「この前」等、過去イベントを参照する文言があるか？
- [ ] そのイベントは全ルートで必ず起きるか？ 起きないルートがあるなら分岐しているか？

## L-007｜メッセージウィンドウの枠からはみ出さない文字数に収める

**修正例**：１ノードのテキストが長すぎて４行となり、メッセージウィンドウの下部からはみ出していた。

**正しい扱い**：
- ノード１つあたり最大３行に収まる文字数（縦長：約90文字、横長：約130文字目安）を守る
- 長い独白や説明は、意味の切れ目でノードを分割する
